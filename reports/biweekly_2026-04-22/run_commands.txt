Run Commands — Quick Reference
================================

=== LOCAL (laptop) ===

# 1. Install dependencies
pip install -r requirements.txt

# 2. Prepare 200-sample subset from CNN/DailyMail
python scripts/01_prepare_subset.py --config configs/subset.yaml

# 3. (Optional) Run judge test in mock mode
python scripts/03_run_judge_test.py --config configs/judging.yaml --n 5


=== MOGON (GPU cluster) ===

# 1. Upload codebase to MOGON
rsync -avz --exclude '.git' --exclude '__pycache__' . muhhas01@hpcgate.zdv.uni-mainz.de:~/thesis/

# 2. SSH to MOGON
ssh mogon

# 3. Set up virtual environment (first time only)
module load lang/Python/3.11.5-GCCcore-13.2.0
module load lib/CUDA/12.1.1
python -m venv ~/venvs/thesis
source ~/venvs/thesis/bin/activate
pip install -r requirements.txt

# 4. Run smoke test (5 samples, ~30 min)
cd ~/thesis
sbatch slurm/smoke_test.sh

# 5. Check smoke test output
cat slurm/smoke_test_<JOBID>.out

# 6. If smoke test passes, run full generation (200 samples, ~2h)
sbatch slurm/generate_candidates.sh

# 7. Run judge test (mock mode, no GPU needed)
sbatch slurm/judge_test.sh

# 8. Evaluate baseline metrics
python scripts/04_evaluate_baseline.py --candidates data/candidates/candidates_200.jsonl

# 9. Generate plots
python scripts/05_plot_results.py --metrics outputs/metrics/baseline_metrics_*.json


=== MONITORING ===

# Check job status
squeue -u muhhas01

# Cancel a job
scancel <JOBID>

# View recent completed jobs
sacct -u muhhas01 --format=JobID,JobName,State,Elapsed,MaxRSS -S 2026-04-22
